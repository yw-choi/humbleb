# HumbleB PRD — Product Requirements Document

## 1. 제품 개요

HumbleB 테니스 클럽(~30명) 운영 웹앱. 매주 정모 참가신청, 대진표 생성, 경기결과 기록을 셀프서비스로 처리.

- **사용자**: 멤버 ~30명, 운영진 2-3명
- **주 사용 환경**: 모바일 (카톡 인앱 브라우저 → 웹앱)
- **핵심 가치**: 운영진 수고 최소화, 멤버 참가 편의성 극대화
- **인프라**: 프론트 Vercel / 백엔드 vesper (FastAPI, nginx HTTPS 프록시) / DB vesper PostgreSQL

---

## 2. 사용자 페르소나

### 멤버 (일반)
- 카톡에서 링크 탭 → 참가 신청 → 정모 당일 대진표 확인 → 경기결과 입력
- 원하는 것: 빠른 참가 신청, 대진표 즉시 확인
- 싫어하는 것: 로그인 반복, 복잡한 UI, 느린 로딩

### 운영진
- 매주 일정 관리, 게스트 등록, 대진표 생성/수정, 카톡에 안내 공유
- 원하는 것: 대진표 자동 생성 + 수동 미세조정, 한 화면에서 모든 관리
- 싫어하는 것: 수작업 대진표 짜기, 엑셀, 별도 관리 앱

## 3. 기능 요구사항

### F1: 카카오 로그인
- **설명**: 카카오 OAuth 2.0으로 인증. 첫 로그인 시 멤버 매핑 1회.
- **수락 기준**:
  - 카카오 로그인 버튼 → OAuth 플로우 → JWT httpOnly 쿠키 세팅
  - 첫 로그인: 멤버 리스트에서 본인 선택 → kakao_id 매핑
  - 이후 로그인: 자동 인식
  - 다른 기기에서도 동일 카카오 계정으로 로그인 가능
  - Redirect URI: `https://vesper.sogang.ac.kr/humbleb/api/auth/kakao/callback`
  - 카톡 인앱 브라우저에서 OAuth 플로우 정상 동작

### F2: 정모 일정 조회
- **설명**: 다가오는 정모 카드 리스트. 상태(멤버신청중/게스트모집/마감), 참가 인원 표시.
- **수락 기준**:
  - 메인 화면에 이번 주 정모 카드 노출
  - 각 카드에 날짜, 시간, 장소, 상태 뱃지, 현재인원/정원
  - 참가/불참 버튼 (인라인, 별도 페이지 이동 없이)
  - 정모 상세 페이지: 참가자 리스트 (이름, 참가시간대, 코멘트)
  - 300ms 내 응답 없으면 스켈레톤 스크린 표시

### F3: 참가 신청/취소
- **설명**: 멤버가 정모에 참가 신청/취소. 참가 시 full/late/early 선택.
- **수락 기준**:
  - 참가 타입: full(풀참), late(시작+30분), early(종료-30분)
  - late + early 동시 선택 불가
  - 참가 타입 선택은 바텀시트로 표시 (기본=full, 변경 원하면 탭)
  - 코멘트 입력 가능 (max 100자)
  - CLOSED 전까지 자유롭게 취소 가능
  - MEMBER_OPEN: 멤버만 신청 가능
  - GUEST_OPEN: 멤버 신청/취소 가능, 게스트는 운영진만 등록
  - 정원 초과 시 신청 불가
  - **Optimistic UI**: 탭 즉시 UI 반영 → 서버 호출 → 실패 시 롤백 + 토스트
  - 탭 디바운스: 300ms

### F4: 참가신청 상태 자동 전환
- **설명**: 일정 상태가 시간에 따라 자동 전환.
- **수락 기준**:
  - 생성 즉시 → MEMBER_OPEN
  - 수요일 15:00 → 정원 미달이면 GUEST_OPEN, 도달이면 CLOSED
  - GUEST_OPEN에서 정원 도달 → CLOSED
  - 정모 당일 → CLOSED (강제)

### F5: 운영진 일정 관리
- **설명**: 일정 CRUD + 반복일정 생성. 별도 앱 없이 같은 앱에서 is_admin 기반 노출.
- **수락 기준**:
  - 일정 생성/수정/삭제
  - 반복일정: weekly 단위로 자동 생성
  - 정원, 코트 수, 장소 등 설정 가능
  - 게스트 등록 (이름, 성별, 실력)
  - 강제 참가/취소 처리
  - 운영진 전용 액션: 오버플로우 메뉴(⋮)로 노출

### F6: 대진표 생성
- **설명**: 참가자 기반으로 대진표 자동 생성. DRAFT → 수정 → CONFIRMED.
- **수락 기준**:
  - 운영진이 라운드 수 + 제약조건 입력 → 대진표 생성
  - DRAFT 상태로 미리보기
  - hard 제약 불충족 시 경고 표시 (누가 몇 게임 적은지)
  - 수동 수정: 드래그앤드롭으로 선수 스왑 (롱프레스 500ms → 드래그)
  - 제약조건 변경 후 재생성 가능
  - 컨펌 → CONFIRMED → 멤버에게 공개
  - CONFIRMED 후에도 수동 수정 가능
  - 대진표 미생성도 허용 (인원 부족 시)

### F7: 제약조건 UI
- **설명**: 운영진이 대진표 생성 시 제약조건을 UI로 추가.
- **수락 기준**:
  - pair_exclude: 같은 팀 금지 (멤버 2명 선택)
  - pair_prefer: 같은 팀 선호 (멤버 2명 선택)
  - opponent_exclude: 상대 매칭 금지 (멤버 2명 선택)
  - round_skip: 특정 라운드 쉼 (멤버 + 라운드)
  - round_start: N라운드부터 참가 (멤버 + 라운드)
  - 복수 제약조건 추가/삭제 가능
  - 제약조건 추가는 바텀시트에서 타입 선택 → 멤버 선택

### F8: 경기결과 입력
- **설명**: 참가자가 경기 스코어 입력.
- **수락 기준**:
  - 매치 참가자 4명 중 아무나 입력 가능
  - 운영진도 입력/수정 가능
  - `inputmode="numeric"` 숫자 키패드, 탭 영역 56×56px
  - 먼저 입력된 것 우선, 이후 수정 가능
  - Optimistic UI: 입력 즉시 반영

### F9: 컨텍스트 어웨어 메인 화면
- **설명**: 시간대 + 참가 여부에 따라 메인 화면 자동 전환.
- **수락 기준**:
  - 평소: 정모 카드 리스트
  - 정모 ±1시간 + 참가자: 대진표 + 결과입력 화면
  - 미참가자: 시간 무관하게 카드 리스트 유지

### F10: 카톡 공유 텍스트 복사
- **설명**: 운영진이 정모 안내 텍스트를 클립보드에 복사 → 카톡에 붙여넣기.
- **수락 기준**:
  - 운영진 페이지에 "안내 복사" 버튼
  - 정모 일시, 장소, 현재 인원, 링크 포함한 텍스트 생성
  - 클립보드 복사 후 토스트 피드백 (2초)

### F11: 대진표 알고리즘
- **설명**: 참가자 기반 자동 대진표 생성. 12명 이하 brute-force.
- **수락 기준**:
  - Hard 제약: 시간 제약, 게임 수 균형(전원 동일, 불가 시 경고), 운영진 제약조건
  - Soft 제약: NTRP 밸런스, 파트너/상대 다양성, 혼복 우선
  - 라운드당 코트 수 × 4명 배정, 나머지 휴식
  - 게스트 실력 매핑: BEGINNER→2.0, INTERMEDIATE→3.0, ADVANCED→4.0

## 4. 비기능 요구사항

### NFR1: 모바일 퍼스트
- 모든 화면 320px~428px 기준 설계
- 터치 타겟 최소 48×48px, 요소 간 최소 간격 8px
- input 폰트 최소 16px (iOS 자동줌 방지)
- 참가 신청 1탭 (로그인 후)
- 뷰포트 단위: `dvh` 사용 (`vh` 금지 — 인앱 브라우저 호환)
- 단일 컬럼 레이아웃, 사이드바 없음

### NFR2: 성능
- 메인 화면 로딩 < 2초 (모바일 4G)
- 대진표 생성 < 3초 (12명, 7라운드)
- API 응답 < 500ms (p95)
- 300ms 초과 시 스켈레톤 스크린 표시

### NFR3: 보안
- JWT httpOnly 쿠키 (XSS 방어)
- 운영진 API는 is_admin 체크
- CORS: Vercel 도메인만 허용

### NFR4: 가용성
- vesper 서버 기반 (대학 인프라)
- DB 백업: cron 기반 pg_dump 주기적 백업
- 30명 동시 접속 처리 가능

### NFR5: 카카오톡 인앱 브라우저 호환
- `position: fixed` 대신 `sticky` 또는 `visualViewport` API
- `window.open()` 사용 금지 → `location.href`
- `alert()`/`confirm()` 사용 금지 → 커스텀 모달/바텀시트
- `localStorage` 불안정 → 인증은 쿠키 기반
- 서비스워커 미지원 → 오프라인 기능 의존 금지
- "브라우저에서 열기" 링크 제공

### NFR6: UI 인터랙션
- Optimistic UI: 참가 신청/취소, 스코어 입력
- 토스트: 에러 4초, 성공 2초, 하단 표시
- 모달 대신 바텀시트 (파괴적 작업만 센터 다이얼로그)
- 바텀시트: 최대 90vh, 상단 핸들바, 배경 스크림 탭으로 닫기
- 로딩: 0-300ms 무표시, 300ms+ 스켈레톤 shimmer

## 5. 범위 외 (Out of Scope)

- 실시간 알림 (푸시, 웹소켓)
- 카카오톡 봇/자동 메시지
- 결제/회비 관리
- 다국어 지원
- 네이티브 앱
- PWA 오프라인 모드 (카톡 인앱 브라우저 서비스워커 미지원)
